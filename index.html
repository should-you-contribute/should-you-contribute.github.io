More About Commits

The number of commits to a repository is a sign of how active a project is. Some projects have dozens of commits per day, while others may only have a commit or two a month. A large number of commits is not inherently good or bad. It depends on whether you're more comfortable with a big, busy project or something smaller and easier to get a handle on.

Projects with no commits in a month or more may be dormant or even abandoned. You should always talk to the community before trying to contribute, but this is especially true for inactive projects.

Keep in mind that not all contributions and activity show up in the git commit history. Publicity, user support, communtiy management and outreach, and other kinds of contribution will not be visible here.
